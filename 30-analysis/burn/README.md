# Burn Series Folder

## What this folder is for
- Stores long-form "burn" working papers that were previously in repo root as `burn_*.md`.
- These files are iterative deep-dive drafts used for high-intensity comparison, rewrite, and synthesis rounds.
- Position in repo taxonomy: this belongs to analysis artifacts, so it lives under `30-analysis`.

## Current contents
- `burn_w1_broad_scan.md`
- `burn_w2_compare.md`
- `burn_w3_counterarguments.md`
- `burn_w4_rewrites.md`
- `burn_w5_question_tree.md`
- `burn_w6_master_digest.md`

## Maintenance rules
- Keep new burn drafts in this folder (do not place `burn_*.md` in root).
- Use naming pattern: `burn_w<week>_<topic>.md`.
- When a burn draft is promoted to a stable deliverable, move/merge it into domain folders (`cross`, `security`, `gateway`, etc.) and keep this folder as the draft history track.
