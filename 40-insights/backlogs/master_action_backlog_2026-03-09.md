# Master Action Backlog（30/60/90）

## 执行顺序（先后与原因）
1. **先做控制面稳定**（网关、观测、策略）  
   - 原因：没有统一控制面，后续 RAG/Agent 优化无法稳定复现。
2. **再做质量与效率优化**（RAG 召回、评测闭环、降本）  
   - 原因：先有可观测和门禁，优化才可量化且可回滚。
3. **最后做规模化治理**（供应链、漂移治理、多栈扩展）  
   - 原因：避免早期复杂化导致交付失败。

---

## 30 天（P0 基线落地）

### A1. 统一推理网关基线（LiteLLM）
- 目标：统一多模型入口、预算与路由策略上线。
- 依赖：K8s namespace、密钥管理、模型清单。
- 工作量：中
- 风险：策略配置不当导致错误路由。
- 验收标准：
  - 至少 3 个 provider 路由可用；
  - 预算超限触发拒绝/降级；
  - 失败自动 fallback 成功率 >= 99%。

### A2. 服务网格熔断护栏（Istio/Envoy）
- 目标：建立网关下游熔断、超时、重试边界。
- 依赖：A1 完成、服务网格基线安装。
- 工作量：中
- 风险：错误阈值配置引发误熔断。
- 验收标准：
  - 注入 5xx 故障后自动 ejection 生效；
  - p95 波动在演练后 30 分钟内回稳。

### A3. 全链路观测（OTel Collector + Grafana/Prometheus）
- 目标：形成 request_id、trace、metric 一体观测。
- 依赖：A1、A2。
- 工作量：中
- 风险：日志字段不一致导致追踪断链。
- 验收标准：
  - trace 覆盖 >= 99%；
  - SLO 仪表盘上线（可用性、时延、成本）；
  - burn-rate 告警可触发。

### A4. 评测门禁（Opik + Promptfoo）
- 目标：把模型/提示词变更纳入 CI 可回归门禁。
- 依赖：A3 指标可用、测试集基线。
- 工作量：中
- 风险：评测集设计不足导致假通过。
- 验收标准：
  - 每次发布前自动跑回归；
  - 失败时自动阻断发布；
  - 输出质量与成本双报告。

### A5. 策略授权控制面（OPA + OpenFGA）
- 目标：高风险工具调用默认拒绝，按策略放行。
- 依赖：工具清单、主体/资源模型定义。
- 工作量：中
- 风险：授权模型设计不当影响业务可用性。
- 验收标准：
  - 未授权高风险动作阻断率 100%；
  - 审计日志可追溯到主体与策略版本。

### A6. 高风险执行隔离（E2B）
- 目标：代码执行/文件系统/网络高风险动作进入沙箱。
- 依赖：A5 策略分类、执行路由改造。
- 工作量：中
- 风险：沙箱资源限额影响任务成功率。
- 验收标准：
  - 高风险工具 100% 沙箱执行；
  - 沙箱逃逸演练 0 成功。

### A7. 向量检索基座（Qdrant）
- 目标：建立默认知识检索层并接入 Agent 主链路。
- 依赖：数据清洗与文档分块规范。
- 工作量：中
- 风险：召回质量不稳定。
- 验收标准：
  - 关键问答集 Top-k 召回指标达标；
  - 检索时延满足 P95 目标。

## 60 天（P1 强化）

### B1. MCP 协议规模化（modelcontextprotocol/servers + python-sdk）
- 目标：形成可管理的 MCP server 目录与接入规范。
- 依赖：A5 审批/授权流程。
- 工作量：中
- 风险：外部 server 供应链风险。
- 验收标准：
  - 所有 server 通过准入清单；
  - 每个 server 有 owner 与 SLA。

### B2. Agent 编排标准化（LangGraph）
- 目标：将核心业务 Agent 全部迁入状态化流程。
- 依赖：A1-A4 完成。
- 工作量：大
- 风险：迁移期逻辑分叉。
- 验收标准：
  - 核心流程可恢复与回放；
  - 线上事故可追溯到节点状态。

### B3. 安全红队自动化（Garak + PyRIT + LLM Guard）
- 目标：建立每周安全回归测试。
- 依赖：A4 门禁管道、A6 沙箱。
- 工作量：中
- 风险：误报过多影响团队效率。
- 验收标准：
  - 每周固定红队报告；
  - 高危问题修复闭环率 >= 95%。

### B4. FinOps 账本层（OpenMeter）
- 目标：网关成本与账本对齐，实现租户级配额控制。
- 依赖：A1 路由日志、A3 指标链路。
- 工作量：中
- 风险：计量口径不一致。
- 验收标准：
  - 网关计量与账本偏差 <= 1%；
  - 超预算自动动作可验证。

## 90 天（P2 规模化与治理）

### C1. 供应链治理（Gitleaks/Semgrep/Cosign）
- 目标：从代码到制品建立强制门禁。
- 依赖：CI/CD 统一流水线。
- 工作量：中
- 风险：流水线时长增加。
- 验收标准：
  - 未签名制品禁止部署；
  - 高危泄露与依赖漏洞 0 漏放。

### C2. 多路线对照试验（Milvus/Weaviate/Portkey/Arize）
- 目标：验证替代路线的成本/性能/治理收益。
- 依赖：A3 统一观测与A4评测门禁。
- 工作量：中
- 风险：试验范围失控。
- 验收标准：
  - 每条替代路线输出统一评分卡；
  - 明确“采纳/观察/淘汰”结论。

### C3. 架构漂移治理
- 目标：按六层建立 owner、ADR、发布前漂移检查。
- 依赖：全部前序行动。
- 工作量：小
- 风险：流程执行流于形式。
- 验收标准：
  - 每次发布必须附 ADR；
  - 漂移报告按月产出并复盘。
