Inference Gateway and Reliability

Why it matters

Gateway design decides provider portability, cost control, and failure containment. A weak gateway turns model outages into business outages.

Key links in this repo

  • Gateway analysis
  • Shortlist and recommendation
  • SLO policy
  • Runbook

Key findings

  • Provider routing must include budget, latency, and policy constraints.
  • Fallback without circuit breakers can amplify cost during outages.
  • Unified request schemas reduce switching risk across providers.

Evidence sources

  • BerriAI/litellm
  • vllm-project/vllm
  • huggingface/text-generation-inference

Risks

  • Policy sprawl if routing rules are scattered across services.
  • Weak metrics makes regression detection too slow.
  • Misconfigured retries can create thundering herds during incidents.

Next actions

  • Define a canonical routing policy schema and ownership.
  • Add budget guardrails per route and per workload.
  • Track route-level SLO and failure signatures weekly.
