RAG Data Stack

Why it matters

RAG quality is constrained by data modeling, chunking, recall metrics, and operational cost. Most failures are data contract failures, not embedding failures.

Key links in this repo

  • Deep dive
  • Data analysis
  • Execution checklist
  • Ecosystem map (CSV)

Key findings

  • Define retrieval quality KPIs before tuning chunk/embedding parameters.
  • Cost models should include index rebuild and data refresh, not just query costs.
  • Evaluation sets must include negative examples and versioned ground truth.

Evidence sources

  • run-llama/llama_index
  • langchain-ai/langchain
  • weaviate/weaviate

Risks

  • Silent relevance regressions without stable eval sets.
  • Index drift when data refresh rules are implicit and untracked.
  • Cost blow-ups from uncontrolled re-embedding and re-indexing.

Next actions

  • Standardize a retrieval-eval pack (queries, docs, labels, metrics).
  • Version the data contract and track refresh cadence explicitly.
  • Gate changes with regression thresholds and rollback plans.
