Agent 编排

为什么重要

生产环境里 Agent 失败往往不是模型不够强，而是编排层缺少恢复、回放与工具边界约束。 对 OwlClaw 这类系统而言，编排质量直接决定稳定性、成本和可审计性。

本仓库入口

  • Agent 研究索引
  • 编排分析
  • 生态地图（CSV）
  • 采纳待办

关键发现

  • 有状态编排通常比纯 prompt 链更抗故障。
  • 多 Agent 不是终点，回放与护栏才是上线门槛。
  • MCP 边界与观测链路应作为默认基线，而非可选项。

证据来源

  • langchain-ai/langgraph
  • openai/openai-agents-python
  • microsoft/autogen
  • crewAIInc/crewAI

主要风险

  • 业务逻辑深绑单框架会导致迁移成本暴涨。
  • 缺少回合护栏时，handoff 循环会放大成本与延迟。
  • trace 契约不完整会让事故追溯失效。

下一步动作

  • 同题双跑并设定明确 stop/go 条件。
  • 新工具接入统一走 MCP 适配层。
  • 上线门禁从“框架指标”升级为“业务结果 + 运行可靠性”。
