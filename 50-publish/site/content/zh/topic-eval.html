门户 English

评测与可观测

为什么重要

没有评测门禁和 trace 覆盖，团队会在“看不见”的状态下上线 prompt 与工具变更， 结果是静默质量漂移和慢事故恢复。

本仓库入口

  • 评测与可观测分析
  • 发布门禁规则（YAML）
  • 回滚手册
  • KPI 目录（CSV）

关键发现

  • trace 覆盖与数据集版本化才是基线，不是模型选择。
  • 评测信号必须绑定晋级规则与回滚触发条件。
  • 日志归一化后，SQL 评测对漂移检测很有效。

证据来源

  • openai/evals
  • langfuse/langfuse
  • truera/trulens

主要风险

  • 指标无 owner 会导致“假自信”。
  • 评测集不刷新会形成采样偏差。
  • 跨工具边界的观测缺口会掩盖回归。

下一步动作

  • 定义晋级门禁：数据 + 评测 + 可靠性 + 成本阈值。
  • 新工具与网关必须满足 trace 契约。
  • 把漂移复盘固定到 release window。
