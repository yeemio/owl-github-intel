推理网关与可靠性

为什么重要

网关决定“可移植性、成本控制、故障隔离”。弱网关会把模型故障升级成业务故障。

本仓库入口

  • 网关分析
  • 候选清单与建议
  • SLO 策略
  • 运行手册

关键发现

  • 路由策略必须纳入预算、延迟与合规约束。
  • 没有断路器的 fallback 会在故障期放大成本。
  • 统一请求模式能降低多供应商切换风险。

证据来源

  • BerriAI/litellm
  • vllm-project/vllm
  • huggingface/text-generation-inference

主要风险

  • 路由规则分散导致策略漂移与不可审计。
  • 指标缺口会拖慢回归发现。
  • 重试/超时策略错误会在事故期触发“羊群效应”。

下一步动作

  • 确立路由策略 schema 与 owner。
  • 为每条路由与每类负载设置预算护栏。
  • 按周追踪 route 级 SLO 与失败签名。
