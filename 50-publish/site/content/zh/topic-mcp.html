MCP 协议与工具互操作

为什么重要

MCP 是把 Agent 与工具实现解耦的工程边界。没有协议边界，框架切换会直接带来重写风险。

本仓库入口

  • MCP 研究索引
  • 生态分析
  • 执行清单
  • 威胁模型

关键发现

  • 一致性与版本纪律比“热门程度”更重要。
  • 发现机制应同时做目录检查与运行期能力检查。
  • 安全姿态差异巨大，必须做 allowlist 与所有权约束。

证据来源

  • modelcontextprotocol/modelcontextprotocol
  • modelcontextprotocol/servers
  • modelcontextprotocol/python-sdk

主要风险

  • 协议合规不等于运行安全。
  • 登记信息与运行行为可能偏离。
  • 如果没有快通道，过严门禁会拖慢紧急集成。

下一步动作

  • 建立传输层与协议版本的兼容性 CI 矩阵。
  • 做风险分层准入（minimal vs strict）并形成明确 verdict。
  • 对 schema 失败和服务降级提供可控 fallback。
