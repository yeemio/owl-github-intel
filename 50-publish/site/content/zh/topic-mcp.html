MCP 生态

Why it matters

MCP 正在从接入协议演化为生态分发层，既影响开发效率也影响治理能力。

Key findings

  • 社区服务器的协议一致性差异较大。
  • Schema 完整度和版本兼容是高频断点。
  • 工具可连通不等于可生产，运行门禁同样关键。

Evidence sources

  • modelcontextprotocol/servers
  • modelcontextprotocol/typescript-sdk
  • wong2/awesome-mcp-servers

Risks

  • Schema 不一致会造成隐蔽行为退化。
  • 协议快速演进易形成兼容性债务。
  • 不同服务器安全与运维成熟度差异显著。

Next actions

  • 建立 MCP 接入质量门槛并强制执行。
  • 维护版本兼容矩阵并纳入日常检查。
  • 为 schema 异常设计降级与回退策略。

MCP 协议与工具互操作

Why it matters

MCP 是 Agent 与工具实现解耦的关键边界。没有协议层，框架切换时会产生高额重写成本。

Key findings

  • 成熟度应看一致性和版本治理，不只看热度。
  • 发现机制要“目录检查 + 运行时能力检查”双轨。
  • Server 安全姿态差异大，必须做白名单和 owner 绑定。

Evidence sources

  • modelcontextprotocol/modelcontextprotocol
  • modelcontextprotocol/servers
  • modelcontextprotocol/python-sdk
  • MCP tools specification

Risks

  • 协议兼容不代表生产安全可控。
  • 目录元数据与真实运行能力可能不一致。
  • 过严门禁会拖慢紧急接入，需例外通道。

Next actions

  • 落地 transport + protocol 兼容性 CI 矩阵。
  • 建立按风险分级的 server 准入流程。
  • 进入 P0 前必须先完成挑战判定与冲突归档。
