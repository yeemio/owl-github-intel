安全与治理

Why it matters

Agent 系统会放大工具调用、上下文注入和策略边界薄弱带来的安全风险。

Key findings

  • 仅靠输入输出检查不够。
  • 工具权限边界与审计轨迹才是核心控制层。
  • 安全控制要在真实多步流程中验证，而非只做静态检查。

Evidence sources

  • guardrails-ai/guardrails
  • protectai/rebuff
  • openai/openai-agents-python

Risks

  • 仅做输出防护会产生“安全假象”。
  • 工具作用域不清可能导致权限升级。
  • 缺少证据链会削弱事后追溯和治理能力。

Next actions

  • 定义并强制执行工具权限策略。
  • 把安全控制纳入发布门禁，而非发布后补救。
  • 在每周挑战循环中跟踪安全回归场景。

门户 English

安全与治理

Why it matters

Agent 平台攻击面覆盖输入、工具调用和生成产物执行。安全治理必须是运行时控制，而不是文档声明。

Key findings

  • 策略引擎、授权图和沙箱应作为一体化控制栈设计。
  • 仅二元通过率不够，必须补对抗覆盖类指标。
  • 供应链信任要同时具备 SBOM、签名和溯源门禁。

Evidence sources

  • open-policy-agent/opa
  • openfga/openfga
  • e2b-dev/E2B
  • sigstore/cosign

Risks

  • 零事故指标可能掩盖检测盲区。
  • 归档或无许可证语料可能带来法律不确定性。
  • 例外通道若无失效机制会绕开策略治理。

Next actions

  • 新增沙箱逃逸演练频率与对抗覆盖指数。
  • 建立有时限的例外审批与签字流程。
  • 发布测试语料与基准引入的法务规则。
